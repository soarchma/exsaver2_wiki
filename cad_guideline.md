# EXSAVER 2.0 웹 관제 시스템 연동을 위한 CAD 도면 작업 표준 가이드

본 가이드는 현장 도면(DXF) 데이터를 기반으로 EXSAVER 2.0 웹 대시보드(BEMS)에 기기 좌표와 연결 상태를 100% 자동 매핑하기 위해 제정된 필수 CAD 작성 규칙입니다.

기존 엑세이버 도면 자동화 프로그램의 룰을 해치지 않으며, 아주 간단한 레이어 및 객체 관리만으로 웹 시스템과의 완벽한 연동을 보장합니다.

## 1. 레이어 및 블록 이름 표준화 (🌟 파싱 키워드)

웹 시스템은 사용자가 그린 객체의 '모양'이 아니라 **'정확한 레이어 이름'과 '블록 이름(Block Name)'**을 추적하여 장비를 자동 인식합니다.

- **요구사항**: 프로젝트 내에서 장비 블록과 배선 레이어의 이름을 통일해야 합니다.

- **배선 레이어**: `EXCELLINE`, `WIRE` 등 사내 표준 준수
- **장비 블록(INSERT)**: LCU(예: `LCU_MAIN`), B/B(예: `SBOX1`, `BUILTIN_BOX`) 등 일관된 뼈대 이름 사용. (임의의 해시 이름 `A$...` 은 내부적으로 자동 투시되므로 뼈대 이름만 유지하면 됩니다.)
- **인수인계**: 도면 작업 완료 후, 이 도면에 사용된 **핵심 레이어/블록 이름 목록을 반드시 웹 시스템 등록 담당자에게 전달**해야 합니다.

## 2. B/B (시스템 박스) 레이어 단독 분리 (필수)

웹 시스템의 파싱 알고리즘이 엉뚱한 건축 기호나 마감선을 B/B로 오인하는 것을 방지하기 위해, B/B 객체는 가급적 독립된 레이어에 존재해야 합니다.

- **요구사항**: B/B 박스는 `3`번 레이어, 또는 `EX_BB` 등 **다른 건축 요소가 섞이지 않은 전용 레이어**에 작성해 주십시오. (불가피하게 섞여 있더라도 블록 이름이 명확하면 시스템이 필터링 가능합니다.)

## 3. B/B 객체 분해(Explode) 엄금 (필수)

B/B 객체가 낱개의 선(LINE) 4개로 분해되어 있으면 시스템이 객체의 고유 이름(Block Name)을 잃어버려 좌표 추출에 실패합니다.

- **요구사항**: B/B 박스는 반드시 **닫힌 다중선(`LWPOLYLINE`)** 형태이거나, 정식 **블록(`BLOCK` / `INSERT`)** 형태로 유지되어야 합니다. 절대 분해(Explode) 명령어를 사용하지 마십시오.

## 4. 웹 파싱용 DXF 추출 (경량화 및 XREF 결합)

웹 시스템의 파싱 엔진이 도면을 1초 만에 분석할 수 있도록, 불필요한 데이터가 제거된 '경량화된 DXF 파일'을 추출해야 합니다.

- **요구사항 1 (엑세이버 요소만 추출)**: 건축/설비 등 불필요한 레이어를 끄고(Off/Freeze), **오직 엑세이버 관련 요소(배선, B/B, LCU, 텍스트)만 화면에 남긴 상태**에서 해당 객체들만 선택하여 DXF로 내보내 주십시오. (`WBLOCK` 명령어 활용)
- **요구사항 2 (외부 참조 결합)**: 타 도면을 참조한 상태라면, 반드시 XREF 관리자 창에서 모든 외부 참조 도면을 **`결합(Bind) ➔ 삽입(Insert)`** 처리하여 단일 도면 내에 완전히 포함시켜 주십시오.

## 5. 프론트엔드 배경용 이미지(PNG) 생성 및 좌표 추출 (필수)

웹 대시보드의 바닥에 깔릴 고해상도 배경 이미지(Base Map)를 CAD 시스템에서 직접 출력하여 DXF와 함께 제공해야 합니다.

### 5.1. PNG 상세 스펙

웹 브라우저에서 확대/축소 시 선명도를 유지하면서도 렌더링 성능을 확보하기 위한 권장 규격입니다.

- **포맷**: `PNG-24` (투명 배경 또는 흰색 배경)
- **해상도 (픽셀)**: 도면의 장변(가장 긴 쪽) 기준 **최소 2000px ~ 최대 4000px** 권장. (6000px 초과 시 브라우저 렌더링 지연 발생)
- **여백(Margin) 제거**: 프린트(PLOT) 설정 시 `Fit to paper` 및 `Center the plot` 옵션을 사용하여 지정한 윈도우 바깥으로 **흰색 여백이 1px도 포함되지 않도록** 출력해야 합니다.

### 5.2. 엑세이버 레이어 숨김 처리

웹 시스템이 동적 아이콘을 도면 위에 직접 그릴 예정이므로, 배경 PNG 도면에는 장비나 배선이 보이면 안 됩니다. 출력 전 **엑세이버 관련 레이어를 모두 숨김(Off) 처리**하여 건축 뼈대만 남겨 주십시오.

### 5.3. 윈도우 좌표 쌍(2개) 추출 (🌟 캘리브레이션 필수 데이터)

배경 이미지의 픽셀과 DXF의 절대 좌표를 웹에서 1:1 매핑하기 위해, PNG 출력 영역(직사각형)의 두 꼭짓점 좌표가 반드시 필요합니다.

1. 도면에 출력할 1개 층 영역을 감싸는 **직사각형 가이드 선(Window Boundary)**을 임시로 그립니다.
1. 해당 사각형의 다음 두 지점의 캐드 절대 좌표(X, Y)를 `LIST` 명령어 등으로 확인합니다.

- **좌측 하단 좌표 (Min X, Min Y)**: 직사각형의 가장 왼쪽 아래 끝점 (예: `X=110000.5, Y=250000.0`)
- **우측 상단 좌표 (Max X, Max Y)**: 직사각형의 가장 오른쪽 위 끝점 (예: `X=185000.0, Y=320000.5`)

## 6. [요약] CAD 작업자 1개 층 처리 시나리오 (Workflow)

실무 작업자가 도면 1개 층의 작업을 마무리하고 시스템 관리자에게 인계하는 순서입니다.

1. **도면 정리 및 결합**: 1층 도면을 엽니다. XREF(외부참조) 도면들을 모두 '결합(Bind)' 처리합니다.
1. **파싱용 DXF 추출**: 엑세이버 관련 레이어만 켠 상태에서 `WBLOCK` 명령어로 객체를 선택해 `1F_data.dxf`로 저장합니다.
1. **PNG 영역 설정 및 좌표 기록**: 엑세이버 레이어를 끄고 건축 뼈대만 켭니다. 도면 전체를 감싸는 **'직사각형 가이드선'**을 그린 후, **'좌측 하단 끝점(Min)'**과 **'우측 상단 끝점(Max)'**의 X, Y 좌표를 메모합니다.
1. **배경용 PNG 출력**: `PLOT` 명령어를 실행합니다. 플롯 영역을 '윈도우(Window)'로 지정하여 가이드선에 스냅을 맞추고, 여백 없이 고해상도로 `1F_bg.png`를 출력합니다.
1. **시스템 직접 업로드 및 설정 입력 (매우 중요)**: EXSAVER 2.0 웹 관리자 페이지에 접속하여 추출된 2개의 파일(`DXF`, `PNG`)을 업로드합니다. 업로드 화면의 입력 폼에 메모해 둔 **좌표 쌍**과 이번 도면에 사용된 **타겟 블록 이름 목록(예: LCU_MAIN, SBOX1 등)**을 직접 입력하고 파싱 작업을 실행합니다.
